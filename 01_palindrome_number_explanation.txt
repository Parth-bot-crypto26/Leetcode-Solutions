===================================================================
LEETCODE 9: PALINDROME NUMBER
===================================================================

[PROBLEM DESCRIPTION]
Given an integer x, return true if x is a palindrome, and false otherwise.
An integer is a palindrome when it reads the same backward as forward.

-------------------------------------------------------------------
[METHODOLOGY: REVERSING THE ENTIRE INTEGER (WITHOUT STRING CONVERSION)]
-------------------------------------------------------------------

1. EDGE CASE HANDLING:
   - Any negative number (e.g., -121) cannot be a palindrome because of the leading negative sign. When reversed, the minus sign ends up at the back ('121-'), which does not match. Thus, we immediately return 'false' if x < 0.

2. PRESERVING THE ORIGINAL VALUE:
   - We need to compare our reversed number with the original value at the end. Since our mathematical operations will progressively destroy the number by dividing it by 10, we make a copy of 'x' into a variable 'num'.

3. MATHEMATICAL REVERSAL LOOP:
   - We run a 'while' loop that continues as long as 'num > 0'.
   - Extract the last digit using the modulo operator: 
     digit = num % 10
   - Append this digit to our running reversed number: 
     rev = (rev * 10) + digit
   - Remove the last digit from our working number using integer division: 
     num = num / 10

4. OVERFLOW PREVENTION:
   - The constraints specify that x can be up to 2^31 - 1. If we reverse a large number, the reversed value might exceed the standard 32-bit signed integer capacity (causing an integer overflow). 
   - To completely safe-guard against this without adding complex logic, we declare 'rev' as a 64-bit 'long long' data type in C++.

5. FINAL COMPARISON:
   - Once the loop terminates, 'rev' contains the completely reversed digits of 'x'.
   - We check if 'rev == x'. If they are equal, it returns true (is a palindrome); otherwise, it returns false.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: x = 121
Initial State: num = 121, rev = 0

Iteration 1:
  - digit = 121 % 10 = 1
  - rev = (0 * 10) + 1 = 1
  - num = 121 / 10 = 12

Iteration 2:
  - digit = 12 % 10 = 2
  - rev = (1 * 10) + 2 = 12
  - num = 12 / 10 = 1

Iteration 3:
  - digit = 1 % 10 = 1
  - rev = (12 * 10) + 1 = 121
  - num = 1 / 10 = 0

Loop terminates (num == 0).
Final Check: rev (121) == x (121) -> Returns TRUE.

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(log10(x))
- Explanation: In every single iteration of the while loop, we divide the number 'num' by 10. Therefore, the total number of operations is directly proportional to the total number of digits in the integer 'x'. 

Space Complexity: O(1)
- Explanation: We are only allocating memory for a few primitive scalar variables ('num', 'rev', and 'digit'). No auxiliary data structures, arrays, or string conversions are used, achieving absolute constant space tracking.
