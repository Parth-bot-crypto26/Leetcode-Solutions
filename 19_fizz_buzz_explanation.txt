===================================================================
LEETCODE 412: FIZZ BUZZ
===================================================================

[PROBLEM DESCRIPTION]
Given an integer 'n', return a 1-indexed string array 'answer' where:
- answer[i] == "FizzBuzz" if i is divisible by both 3 and 5.
- answer[i] == "Fizz" if i is divisible strictly by 3.
- answer[i] == "Buzz" if i is divisible strictly by 5.
- answer[i] == i (as a string) if none of the above conditions are true.

-------------------------------------------------------------------
[METHODOLOGY: CONDITIONAL LADDER SIMULATION]
-------------------------------------------------------------------

The solution uses a single loop to iterate from 1 to 'n', mapping each number 
to its corresponding string value based on divisibility rules.

1. CRITICAL ORDER OF CONDITIONS:
   - The ordering of the 'if-else' statements is the single most important 
     factor in this problem. Numbers divisible by both 3 and 5 (multiples of 15) 
     are automatically divisible by 3 individually and 5 individually.
   - Therefore, the compound check 'i % 3 == 0 && i % 5 == 0' must be evaluated 
     first. If a single check like 'i % 3 == 0' came first, a number like 15 
     would be incorrectly intercepted, evaluated as "Fizz", and the remaining 
     checks would be skipped.

2. LOGICAL FALL-THROUGH:
   - If the number is not a multiple of 15, the program falls through to check if 
     it is exclusively a multiple of 3 ('Fizz') or 5 ('Buzz').
   - If all divisibility conditions fail, we convert the raw loop index 'i' into 
     its string representation using C++'s built-in 'to_string(i)' method.

3. MEMORY OPTIMIZATION NOTE:
   - While your current initialization 'vector<string> result;' works flawlessly, 
     a common production optimization is pre-allocating the vector size up front 
     using 'vector<string> result(n);' to prevent internal dynamic array reallocation 
     and copying overhead during 'push_back()' operations.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: n = 5
Initial State: result = []

- i = 1: 1%3==0 && 1%5==0 (F) -> 1%3==0 (F) -> 1%5==0 (F) -> else: push_back("1")
- i = 2: 2%3==0 && 2%5==0 (F) -> 2%3==0 (F) -> 2%5==0 (F) -> else: push_back("2")
- i = 3: 3%3==0 && 3%5==0 (F) -> 3%3==0 (T) -> push_back("Fizz")
- i = 4: 4%3==0 && 4%5==0 (F) -> 4%3==0 (F) -> 4%5==0 (F) -> else: push_back("4")
- i = 5: 5%3==0 && 5%5==0 (F) -> 5%3==0 (F) -> 5%5==0 (T) -> push_back("Buzz")

Loop terminates.
Returns result = ["1", "2", "Fizz", "4", "Buzz"]

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(n)
- Explanation: We run a single linear loop from 1 to 'n'. Within each iteration, 
  performing basic modulo operations and pushing a string element into the vector 
  runs in constant O(1) time. Thus, the total processing time scales linearly 
  with the value of 'n'.

Space Complexity: O(1) [Auxiliary Space]
- Explanation: Excluding the memory allocated for the 'result' vector itself 
  (which is strictly required as the output interface), the algorithm runs 
  entirely in-place. No extra auxiliary arrays, maps, or recursive stack frames 
  are generated, resulting in constant auxiliary space complexity.
