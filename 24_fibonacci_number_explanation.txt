===================================================================
LEETCODE 509: FIBONACCI NUMBER
===================================================================

[PROBLEM DESCRIPTION]
The Fibonacci numbers form a sequence where each number is the sum of the 
two preceding ones, starting from 0 and 1.
Formally:
  F(0) = 0, F(1) = 1
  F(n) = F(n - 1) + F(n - 2), for n > 1.

Given 'n', calculate F(n).

-------------------------------------------------------------------
[METHODOLOGY: ITERATIVE STATE-SLIDING (DYNAMIC PROGRAMMING)]
-------------------------------------------------------------------

A basic recursive solution (F(n-1) + F(n-2)) breaks down into an exponential 
tree of calculations, resulting in a disastrous O(2^n) time complexity due to 
repeatedly recalculating identical subproblems. 

Your solution leverages a bottom-up iterative approach (Tabulation concept) but 
optimizes the space footprint down to absolute constant memory by recognizing that 
we only ever need the last *two* values to compute the next state.

1. BASE CLAUSES:
   - If n == 0, return 0.
   - If n == 1, return 1.

2. RUNNING STATE POINTERS:
   - We initialize 'f1 = 0' (representing F(i-2)) and 'f2 = 1' (representing F(i-1)).

3. THE STATE-SLIDING LOOP:
   - We run a loop from 'i = 2' up to 'n'.
   - At each step:
     * Compute the current Fibonacci value: 'fi = f1 + f2'.
     * Shift our historical reference point forward: 'f1' takes the old value of 'f2'.
     * Shift our leading value forward: 'f2' takes the newly calculated value of 'fi'.

4. EXTRACTION:
   - Once the loop reaches 'n', 'f2' holds the exact computed value of F(n), which 
     we return.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: n = 4 (Expected Output: 3)
Initial State: f1 = 0, f2 = 1

- i = 2: fi = f1 + f2 -> 0 + 1 = 1
         Update history: f1 = 1, f2 = 1
- i = 3: fi = f1 + f2 -> 1 + 1 = 2
         Update history: f1 = 1, f2 = 2
- i = 4: fi = f1 + f2 -> 1 + 2 = 3
         Update history: f1 = 2, f2 = 3

Loop terminates because i exceeds n.
Returns f2 -> 3.

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(n)
- Explanation: The algorithm uses a single loop that runs from 2 up to 'n'. 
  Inside the loop, only basic arithmetic addition and assignment calculations are 
  performed, executing in O(1) time. The runtime scales completely linearly 
  with the input size.

Space Complexity: O(1)
- Explanation: Rather than saving a full state table array of size 'n' 
  (which standard dynamic programming tabulation does), this sliding window approach 
  overwrites the same three integer tracking variables ('f1', 'f2', 'fi') in memory, 
  preserving constant auxiliary space.
