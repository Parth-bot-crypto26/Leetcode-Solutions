===================================================================
LEETCODE 1929: CONCATENATION OF ARRAY
===================================================================

[PROBLEM DESCRIPTION]
Given an integer array 'nums' of length 'n', create an array 'ans' of 
length '2n' where ans[i] == nums[i] and ans[i + n] == nums[i] for 0 <= i < n.

Specifically, 'ans' is the concatenation of two 'nums' arrays.

-------------------------------------------------------------------
[METHODOLOGY: PRE-ALLOCATED DUAL-INDEX SINGLE-PASS MAPPING]
-------------------------------------------------------------------

The naive way to handle array concatenation is to copy elements sequentially 
using two separate loops or pushing elements one by one. We can maximize 
performance by leveraging static sizing and a dual-pointer mapping offset.

1. MEMORY PRE-ALLOCATION (CAPACITY OPTIMIZATION):
   - When vectors grow dynamically via consecutive push-backs without a predefined 
     size, the underlying runtime engine must repeatedly reallocate memory blocks 
     and copy elements over whenever it hits its current capacity limit.
   - By explicitly defining the constructor size to twice the input magnitude 
     'vector<int> ans(2 * n);', we request a single contiguous memory block 
     upfront, converting vector writes into instant raw index assignments.

2. EQUIDISTANT COMPONENT TRANSPOSE (i + n):
   - The output structure requires the original pattern to repeat immediately after 
     the first 'n' slots.
   - Using a single linear loop from 'i = 0' to 'n - 1', we populate two slots 
     simultaneously during each step:
     * First half placement: ans[i] = nums[i]
     * Second half mirror placement: ans[i + n] = nums[i]

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: nums = [1, 2, 1] | n = 3
Initial State: ans allocated with size 6 -> [0, 0, 0, 0, 0, 0]

- i = 0: ans[0] = nums[0] -> ans[0] = 1
         ans[0 + 3] = nums[0] -> ans[3] = 1
- i = 1: ans[1] = nums[1] -> ans[1] = 2
         ans[1 + 3] = nums[1] -> ans[4] = 2
- i = 2: ans[2] = nums[2] -> ans[2] = 1
         ans[2 + 3] = nums[2] -> ans[5] = 1

Loop terminates.
Returns ans = [1, 2, 1, 1, 2, 1]

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(n)
- Explanation: The algorithm uses a single loop that runs exactly 'n' times, where 
  'n' is the length of the input array. Within the loop body, it executes exactly 
  two simple array assignment statements, operating in O(1) constant time per loop step. 
  The total processing speed scales strictly linearly.

Space Complexity: O(1) [Auxiliary Space]
- Explanation: Excluding the memory allocated for the 'ans' array itself, which is 
  an explicit requirement of the output return interface, the algorithm uses no 
  additional data structures, memory vectors, or recursive tracking states.
