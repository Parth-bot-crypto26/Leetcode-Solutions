===================================================================
LEETCODE 1022: SUM OF ROOT TO LEAF BINARY NUMBERS
===================================================================

[PROBLEM DESCRIPTION]
You are given the root of a binary tree where each node has a value 0 or 1. 
Each root-to-leaf path represents a binary number starting with the most 
significant bit. For all leaves in the tree, consider the numbers represented 
by the path from the root to that leaf. Return the sum of these numbers.

-------------------------------------------------------------------
[METHODOLOGY: PRE-ORDER DEPTH FIRST SEARCH (DFS) WITH BITWISE CASCADE]
-------------------------------------------------------------------

To evaluate binary values representing paths from root to leaf, we must build the 
number sequentially from the most significant bit (MSB) down to the least 
significant bit (LSB). 

Instead of passing an expensive string or list of bits down the recursive call stack, 
we can dynamically compute the running decimal value of the binary path on the fly 
using standard bitwise operations.

1. THE BIT SHIFT ACCUMULATION LAW:
   - When we transition from a parent node down to a child node, every bit we have 
     seen so far shifts left by exactly 1 position (equivalent to multiplying the 
     decimal sum by 2).
   - We execute this state change instantly using: 'currentSum = (currentSum << 1) | node->val;'
   - The bitwise OR ('|') places the current node's binary value into the newly vacated 
     least significant bit position.

2. BASE LEAF CONDITIONAL TRIGGER:
   - If a node has no left or right children ('!node->left && !node->right'), it is 
     a valid leaf node. The 'currentSum' variable at this execution depth now holds the 
     completed numerical value for that unique path. We return it immediately.

3. DIVIDE AND CONQUER SUMMATION:
   - For internal branching nodes, we recursively explore both paths by forwarding 
     the updated 'currentSum' to the left child and right child branches.
   - We sum the integer results returned from both subtrees and bubble the consolidated 
     value back up to the caller stack frame: 'return dfs(left) + dfs(right);'

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input Tree: [1, 0, 1] 
       1 (Root)
      / \
     0   1 (Leaves)

- Depth 1 (Root, node->val = 1):
  * currentSum = (0 << 1) | 1 = 1
  * Internal node -> Recurse Left and Right.

- Depth 2 (Left Leaf, node->val = 0):
  * currentSum = (1 << 1) | 0 -> 2 | 0 = 2 (Binary: 10)
  * Leaf triggered! Returns 2.

- Depth 2 (Right Leaf, node->val = 1):
  * currentSum = (1 << 1) | 1 -> 2 | 1 = 3 (Binary: 11)
  * Leaf triggered! Returns 3.

Bubble Up Phase:
  - Root receives 2 from left and 3 from right.
  - Total Sum = 2 + 3 = 5.

Returns 5.

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(N)
- Explanation: Where 'N' is the total number of nodes in the binary tree. 
  The depth-first search traversal algorithm visits every single node exactly once. 
  Within each node frame, it executes O(1) bit shifts, state assignments, and 
  logical boundary checks, running in optimal linear time.

Space Complexity: O(H)
- Explanation: Where 'H' is the height of the binary tree. Memory footprint is strictly 
  bound by the maximum depth of the recursive call stack. In the worst-case scenario 
  of a completely skewed tree, 'H' equals 'N' (O(N) space). In a perfectly balanced 
  binary tree, 'H' equals log(N), achieving highly optimized O(log N) stack usage.
