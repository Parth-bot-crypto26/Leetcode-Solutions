===================================================================
LEETCODE 1103: DISTRIBUTE CANDIES TO PEOPLE
===================================================================

[PROBLEM DESCRIPTION]
We distribute a given number of 'candies' to a row of 'num_people' in an 
incremental, cyclic fashion. We give 1 candy to the first person, 2 to the second, 
up to 'n' to the last person. Then we loop back to the start, giving 'n + 1' candies 
to the first person, 'n + 2' to the second, and so on.

This repeats, incrementing the distribution requirement by 1 each turn, until 
the candy supply runs completely dry. The final person receives whatever residual 
candies remain. Return an array logging the exact total candies received by each person.

-------------------------------------------------------------------
[METHODOLOGY: GREEDY SUPPLY DEPLETION & CYCLIC INDEX MODULO SCAN]
-------------------------------------------------------------------

Instead of over-complicating this with quadratic mathematical series derivations, 
a clean dynamic simulation loop easily runs within performance limits. Given that 
the required candy batch grows linearly ($1 + 2 + 3 + 4 + ...$), the candy supply 
evaporates incredibly fast—meaning our simulation loop will hit its stop condition 
in O(sqrt(candies)) operations.

1. CAPACITY ALLOCATION:
   - We construct a response vector 'ans' pre-allocated to size 'num_people', 
     initialized with zeros.

2. INCREMENTAL CYCLIC SIMULATION LOOP:
   - We run a loop while 'candies > 0'.
   - To figure out what the active person should receive, we take the minimum 
     between our current target amount and whatever inventory remains:
     'to_give = min(candies, current_give)'.
   - We credit this to the active person 'ans[i] += to_give' and deduct it from 
     our master inventory tracking pool 'candies -= to_give'.

3. MODULO TRAVERSAL & STATE SHIFTING:
   - We increment our 'current_give' value to prepare for the next turn.
   - We advance our index pointer circularly using a basic modulo wrapping expression:
     'i = (i + 1) % num_people'. This seamlessly snaps our index back to 0 whenever 
     we overshoot the rightmost edge of the row array.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: candies = 7, num_people = 4
Initial State: ans = [0, 0, 0, 0], i = 0, current_give = 1

- Iteration 1 (i = 0): to_give = min(7, 1) = 1.
                      ans[0] += 1 -> [1, 0, 0, 0] | candies = 6
                      current_give = 2 | i = (0 + 1) % 4 = 1

- Iteration 2 (i = 1): to_give = min(6, 2) = 2.
                      ans[1] += 2 -> [1, 2, 0, 0] | candies = 4
                      current_give = 3 | i = (1 + 1) % 4 = 2

- Iteration 3 (i = 2): to_give = min(4, 3) = 3.
                      ans[2] += 3 -> [1, 2, 3, 0] | candies = 1
                      current_give = 4 | i = (2 + 1) % 4 = 3

- Iteration 4 (i = 3): to_give = min(1, 4) = 1 (Residual Supply Exhausted).
                      ans[3] += 1 -> [1, 2, 3, 1] | candies = 0
                      current_give = 5 | i = (3 + 1) % 4 = 0

Loop terminates safely because candies == 0.
Returns [1, 2, 3, 1].

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(max(num_people, sqrt(candies)))
- Explanation: The number of loops is bound by how fast the triangular sum of 
  candies ($k \cdot (k + 1) / 2$) crosses our total inventory threshold. For a maximum 
  candy count of 10^9, 'k' scales to roughly 45,000 checks, executing almost instantly. 
  The overall performance window is highly optimal.

Space Complexity: O(1) [Auxiliary Space]
- Explanation: Excluding the final output storage container requested by the problem 
  interface, memory allocation is locked to primitive tracking scalar parameters 
  ('i', 'current_give', 'to_give'), guaranteeing absolute constant space.
