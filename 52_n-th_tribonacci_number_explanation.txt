===================================================================
LEETCODE 1137: N-TH TRIBONACCI NUMBER
===================================================================

[PROBLEM DESCRIPTION]
The Tribonacci sequence Tn is defined as follows: 
T0 = 0, T1 = 1, T2 = 1, and Tn+3 = Tn + Tn+1 + Tn+2 for n >= 0.

Given 'n', return the value of Tn. The constraints state 0 <= n <= 37, 
and the output is guaranteed to fit within a standard 32-bit signed integer.

-------------------------------------------------------------------
[METHODOLOGY: THREE-POINTER SLIDING WINDOW DYNAMIC PROGRAMMING]
-------------------------------------------------------------------

A basic recursive implementation of a three-term sequence creates an exponential 
O(3^N) call tree, leading to severe performance failure due to repetitive subtree 
reevaluations. We can completely optimize this by using an iterative bottom-up 
approach.

Furthermore, calculating a term strictly requires knowing the values of the **three 
preceding numbers**. Keeping an entire table array of size N is a waste of memory. 
We can optimize our space footprint down to O(1) constant space using a rolling 
sliding window state replacement system.

1. EDGE CASE BASE IMPLEMENTATION:
   - We implement short-circuit conditions at the top of the execution flow to handle 
     the initial seed base states:
     * If n == 0, return 0.
     * If n == 1 or n == 2, return 1.

2. STATE SHIFTING TRAVERSAL:
   - We initialize our state variables to match the base cases: t0 = 0, t1 = 1, t2 = 1.
   - We run a linear loop starting from index 3 up to 'n':
     * We calculate the next number in the sequence by adding the previous three terms:
       tn = t0 + t1 + t2
     * We shift our window one position forward by reassigning our state variables:
       t0 takes the value of t1
       t1 takes the value of t2
       t2 takes the newly calculated value of tn
   - When the loop finishes, 't2' holds the completed evaluation value for position 'n'.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: n = 4
Initial State: t0 = 0, t1 = 1, t2 = 1

- i = 3: tn = t0 + t1 + t2 -> 0 + 1 + 1 = 2
         Shift: t0 = 1 | t1 = 1 | t2 = 2

- i = 4: tn = t0 + t1 + t2 -> 1 + 1 + 2 = 4
         Shift: t0 = 1 | t1 = 2 | t2 = 4

Loop terminates safely (i matches n).
Returns t2 -> 4.

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(n)
- Explanation: The algorithm processes the sequence values linearly using a single loop 
  that runs exactly 'n - 2' times. Each step performs a fixed sequence of arithmetic 
  additions and pointer variable shifts in O(1) constant time, making the runtime scale 
  completely linearly.

Space Complexity: O(1)
- Explanation: Dynamic memory allocation arrays are completely bypassed. The state tracking 
  history is managed entirely inside four local primitive integer registers on the stack 
  ('t0', 't1', 't2', 'tn'), ensuring absolute constant space overhead.
