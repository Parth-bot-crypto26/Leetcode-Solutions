===================================================================
LEETCODE 1232: CHECK IF IT IS A STRAIGHT LINE
===================================================================

[PROBLEM DESCRIPTION]
You are given an array 'coordinates', where coordinates[i] = [x, y] 
represents a point on the X-Y plane. Check if all these points form a 
perfectly straight line.

-------------------------------------------------------------------
[METHODOLOGY: ANCHORED VECTOR CROSS-MULTIPLICATION SCAN (O(N))]
-------------------------------------------------------------------

To determine if a sequence of points is collinear, the slope between any two consecutive 
points must remain completely constant across the entire set. 

THE RAW DIVISION REFACTOR:
The standard slope formula between two coordinates is given by:
  Slope = (y1 - y0) / (x1 - x0) = Δy / Δx

If we were to calculate raw slopes explicitly using floating-point operations, our 
program would break under vertical alignments where Δx = 0 (throwing a fatal 
divide-by-zero runtime panic). It would also fall victim to decimal rounding inaccuracies.

THE CROSS-PRODUCT INTEGER METHOD:
We eliminate division entirely by multiplying out the denominators crosswise.
1. We establish our baseline directional reference vector using the first two points:
   - dx = x1 - x0
   - dy = y1 - y0
2. For every subsequent point (x, y) starting from index 2, the slope relative to the 
   anchor point (x0, y0) must match our baseline:
   - (y - y0) / (x - x0) == dy / dx
3. Cross-multiplying this equation leaves us with a highly optimal, pure integer 
   comparison guard statement:
   - dy * (x - x0) == dx * (y - y0)

If any coordinate fails this condition, it breaks collinearity alignment, and we 
short-circuit to return false immediately.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------

Input: coordinates = [[1,1], [2,2], [3,4]]
Anchors:
  - (x0, y0) = (1, 1)
  - (x1, y1) = (2, 2)
  - dx = 2 - 1 = 1
  - dy = 2 - 1 = 1

Iteration 1 (i = 2): Point = (3, 4)
  - Left Side:  dy * (x - x0) -> 1 * (3 - 1) = 2
  - Right Side: dx * (y - y0) -> 1 * (4 - 1) = 3
  - Check: Is 2 == 3? FALSE! 

Condition fails, short-circuiting to return false.
Returns false.

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(n)
- Explanation: Where 'n' is the total number of points inside the 'coordinates' vector. 
  The algorithm iterates through the collection exactly once, evaluating a fixed set 
  of primitive integer math subtractions and multiplications running in O(1) constant 
  time per element.

Space Complexity: O(1)
- Explanation: The calculation operates completely in-place. Auxiliary storage is 
  restricted to local scalar primitive registers tracking displacements and indices 
  on the stack, ensuring an absolute constant memory footprint.
