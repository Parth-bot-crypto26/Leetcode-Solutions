===================================================================
LEETCODE 1342: NUMBER OF STEPS TO REDUCE A NUMBER TO ZERO
===================================================================

[PROBLEM DESCRIPTION]
Given an integer num, return the number of steps to reduce it to zero. 
In one step, if the current number is even, you have to divide it by 2; 
otherwise, you have to subtract 1 from it.

-------------------------------------------------------------------
[METHODOLOGY: BITWISE ALGEBRA & HARDWARE REG-LEVEL POPCOUNT (O(1))]
-------------------------------------------------------------------

Your iterative solution perfectly models the problem statement. However, 
the transition steps (divide by 2 if even, subtract 1 if odd) correspond 
exactly to native low-level bitwise operations:
  - 'num /= 2' is identical to a Bitwise Right Shift ('num >> 1').
  - 'num -= 1' on an odd number flips the lowest active bit from 1 to 0.

THE BINARY REPRESENTATION PROPERTY:
Consider the number 14 in binary: '1110'.
  - Bit is 0 (Even) -> Shift right -> leaves '111' (Costs 1 step)
  - Bit is 1 (Odd)  -> Flip to 0   -> leaves '110' (Costs 1 step)
  - Bit is 0 (Even) -> Shift right -> leaves '11'  (Costs 1 step)
  - Bit is 1 (Odd)  -> Flip to 0   -> leaves '10'  (Costs 1 step)
  - Bit is 0 (Even) -> Shift right -> leaves '1'   (Costs 1 step)
  - Bit is 1 (Odd)  -> Flip to 0   -> leaves '0'   (Costs 1 step)
Total steps = 6.

THE DIRECT COMBINATORIAL FORMULA:
By analyzing this pattern, the total number of operations required is:
  Total Steps = (Total bits from the highest '1' bit to the end) + (Count of all '1' bits) - 1

Instead of checking bits one by one using a slow loop, modern C++ compilers 
expose direct hardware intrinsic functions that calculate these bit states 
in a single CPU clock cycle:
  1. 'std::popcount(num)': Counts the total number of set '1' bits.
  2. 'std::countl_zero(num)': Counts the number of leading zero bits up to 
     the highest '1' bit. In a 32-bit integer, the length of the number is 
     calculated by: '32 - std::countl_zero(num)'.

This allows us to evaluate the exact answer using pure register arithmetic 
without a single loop or branch condition.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: num = 14 (Binary: 00000000 00000000 00000000 00001110)

- Step 1: Count leading zeros
  countl_zero(14) = 28.
  Total bit length = 32 - 28 = 4 bits.

- Step 2: Count set bits (Population Count)
  popcount(14) = 3 (since bits are at 8, 4, and 2).

- Step 3: Compute Formula
  Steps = (Total bit length - 1) + popcount
        = (4 - 1) + 3 
        = 3 + 3 = 6.

Returns 6.

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(1)
- Explanation: By using hardware-level bit count intrinsics, the algorithm 
  completely eliminates loops and conditional branch paths. The entire 
  calculation runs inside fixed, single-cycle CPU instructions, providing 
  unbeatable constant execution time.

Space Complexity: O(1)
- Explanation: The operation runs entirely inside active CPU registers, 
  avoiding any stack allocation or variable definitions.
