===================================================================
LEETCODE 3741: MINIMUM DISTANCE BETWEEN THREE EQUAL ELEMENTS II
===================================================================

[PROBLEM DESCRIPTION]
Given an integer array 'nums', find a good tuple (i, j, k) of 3 distinct indices 
such that nums[i] == nums[j] == nums[k]. 

The distance of a good tuple is defined as:
  abs(i - j) + abs(j - k) + abs(k - i)

Return the minimum possible distance among all valid good tuples. If no good 
tuples exist, return -1.

-------------------------------------------------------------------
[METHODOLOGY: ALGEBRAIC CANCELLATION & SLIDING TRIPLET WINDOW]
-------------------------------------------------------------------

A brute-force evaluation of all index combinations takes O(N^3) time. However, 
algebraic simplification reduces this problem to a linear scan.

1. DISTANCE FORMULA REDUCTION:
   Assume without loss of generality that three indices are sorted: i < j < k.
   Expanding the absolute value terms gives:
     abs(i - j) = j - i
     abs(j - k) = k - j
     abs(k - i) = k - i
   Summing them up:
     Distance = (j - i) + (k - j) + (k - i)
              = j - i + k - j + k - i
              = 2k - 2i
              = 2 * (k - i)

   Crucially, the middle index 'j' cancels out completely! The distance is 
   strictly dependent on the gap between the first and last indices of the triplet.

2. ADJACENT TRIPLET INVARIANT:
   - For any value appearing at sorted indices [pos[0], pos[1], ..., pos[m-1]], 
     to minimize 2 * (pos[k] - pos[i]) where k - i >= 2, we should choose indices 
     that are as close as possible in the array.
   - The minimum distance for a value will always occur between adjacent triplet 
     indices: pos[i] and pos[i + 2].

3. ALGORITHM STEPS:
   - Group all occurrence indices by element value using a hash map or array.
   - For each element with at least 3 occurrences, iterate through adjacent triplets 
     (pos[i], pos[i+2]) and compute 2 * (pos[i+2] - pos[i]).
   - Return the minimum distance found across all values, or -1 if no element 
     appears 3 or more times.

-------------------------------------------------------------------
[DRY RUN EXAMPLE]
-------------------------------------------------------------------
Input: nums = [1, 2, 1, 1, 3]

Step 1: Group Indices by Value
  - Value 1: indices = [0, 2, 3]
  - Value 2: indices = [1]
  - Value 3: indices = [4]

Step 2: Process Triplets
  - Value 1 has size 3 >= 3:
    * i = 0: first = pos[0] = 0, third = pos[2] = 3
    * current_dist = 2 * (3 - 0) = 6
    * min_dist = 6

Returns 6.

-------------------------------------------------------------------
[COMPLEXITY ANALYSIS]
-------------------------------------------------------------------

Time Complexity: O(n)
- Explanation: Grouping indices into lists takes O(n) time. Iterating over 
  all stored indices to check adjacent triplets takes O(n) total time across 
  all keys, yielding a overall linear runtime.

Space Complexity: O(n)
- Explanation: The hash map stores a total of 'n' indices across all vectors, 
  requiring O(n) auxiliary memory.
